# Лабораторна робота №4 — Тестування

## 1. Unit та інтеграційні тести
Використано `pytest` та `httpx` для тестування API з Лабораторної №2.
Запуск тестів з перевіркою покриття (code coverage):
`pytest lab4/test_api.py --cov=lab2.main --cov-report=term-missing`

## 2. Performance-тестування
Використано `locust`.
Складний сценарій: створення статті (POST), отримання її `id` з відповіді, та наступне видалення цієї статті (DELETE) за отриманим ідентифікатором.
Запуск: `locust -f lab4/locustfile.py`

## 3. Автоматичний скрапінг
Використано `selenium`.
Сценарій: відкриття локальної сторінки Лабораторної №3, введення логіну та паролю, натискання кнопки, очікування завантаження статей та їх зчитування.
Запуск: `python lab4/scraper.py`
